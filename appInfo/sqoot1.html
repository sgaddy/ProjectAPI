Sqoot API

CORS (Cross-Origin Resource Sharing)

The API supports CORS for AJAX requests as a way of getting around same origin policy. That means you can make cross-domain requests to the API without having to use JSONP. CORS is just special response headers sent from the server that tells the client (the browser) that cross-domain requests are allowed, so it doesn’t require any extra steps client-side to make it work. You just make your AJAX requests as normal. CORS is supported by most modern browsers—in some cases it’s been supported for many versions back—but if you need to support legacy browsers you’ll need to use JSONP.

JSONP

We has it. If you need to hit our API from the client and need to support legacy browsers, you’ll need to use JSONP to get around same origin policy.

Time formatting

There’s just no great way to represent time in JavaScript. We decided to represent all times as human-readable, ISO 8601 strings in UTC. So January 24th, 2013 at 1:00PM would look be “2012-01-24T13:00:00Z”.
